Прототип повторяет соответствующую функциональность Куба (MDR32Cube) для корпуса BGA144. Графический инструмент состоит из трех элементов. Сканера монтажных площадок корпуса микросхемы. Набора кнопок для выбора функционального назначения ножки микроконтроллера. И двухкаскадного фильтра для подсветки ножек с заданным функциональным назначением.

void Delay(int waitTicks)
{
    int i;
    for (i = 0; i < waitTicks; i++)
    {
        __NOP();
    }
}

Проведите указатель мыши над ножками микросхемы. Правее чертежа, на кнопках, будут меняться надписи – это список из 16 функций, любая из которых может быть закреплена за ножкой под курсором. Найти требуемую функцию может быть затруднительно. Полный список содержит более 1500 позиций (144 x 16). Решить задачу помогает фильтр. Проведите указатель мыши над второй колонкой кнопок с названиями периферийных устройств микроконтроллера. Выберите таймеры (TMR). Они подключены ко всем конфигурируемым ножкам. Проведите указатель мыши над второй колонкой кнопок фильтра – группы подсвечиваемых ножек будут меняться. Выберите нулевой таймер – вы активировали второй каскад фильтра, который может иметь несколько критериев поиска объединяемых логическим оператором "И". Перемещайте курсор мыши – убедитесь в том, что у нулевого таймера есть четыре канала захвата / сравнения. Проверьте, что все каналы имеют комплементарные входы / выходы.

Для определения функционального назначения за ножкой микроконтроллера подведите к ней указатель мыши. Щёлкните левую кнопку – справа от микросхемы активируются 16 кнопок с именами функций. Выберите нужную и щёлкните по ней. Будет сгенерирован листинг программы на языке Си. Он появиться на месте этого текста. Продолжите процедуру определения функционального назначения для других ножек – листинг программы будет меняться.

После выбора функции, зажав клавишу Shift, с помощью указателя мыши можно уточнить тип вывода (вход, двухтактный выход, выход с открытым стоком или с открытым истоком). Условные графические изображения типа вывода следует интерпретировать по следующим правилам. Иконка аналогового входа совпадает УГО АЦП (вход – слева). Иконка цифрового входа совпадает с УГО логического элемента (вход – слева). Аналоговый выход обозначается так же как ЦАП (выход – справа). Иконки цифровых выходов – стандартные – ромбы (для двухтактного оконечного каскада, для каскада с открытым стоком / коллектором, для каскада с открытым истоком / эмиттером). Незадействованные выводы обозначены кружками. Выводы земли и питания – квадратиками. Зажав комбинацию клавиш Shift + Ctrl можно включить резистор подтяжки потенциала к питанию или к земле.

Описание программы

Программа является автономной веб-страницей. Сторонние библиотеки не задействованы. Код приложения обращается исключительно к прикладным программным интерфейсам браузера. Пользователь может сохранить данный документ на локальный диск (Ctrl+S) с фильтром "Веб-страница, только HTML" – программа сохранит функциональность (без сервера).

Приложение, условно, можно разделить на четыре части. Первая часть – это внедрённый в веб-страницу csv-файл (comma-separated values) с данными из спецификации на микроконтроллер [1]. Содержимое соответствующего тега можно скопировать в табличный процессор и воспользоваться инструментом "Текст по столбцам" для анализа. Вторая часть приложения – это скрипт, поддерживающий взаимодействие с пользователем средствами интерактивной формы. Скрипт не простой – обслуживает большое количество состояний и не поддаётся разбиению на модули. Поэтому, каждый раз, в ответ на выбор пользователя, создаёт программное событие, пакует данные и передаёт их третьей части приложения. Это скрипт, в отдельном теге, решающий задачу генерации Си-кода. Поддерживает генерацию кода четвёртая часть приложения – внедрённый в страницу файл исходного кода на языке Си, конфигурирующий устройства микроконтроллера. Он имеет линейную структуру и разбит на фрагменты, в которых упоминаются псевдо-имена устройств. В зависимости от выбора пользователя, управляющий кодогенерацией скрипт включает в итоговый листинг необходимые фрагменты, в нужном количестве, выполняя замену псевдо-имён на фактические. Безусловно, большая часть устройств ввода-вывода может функционировать в разных режимах. Но структура шаблонов конфигурирующего кода от этого не меняется. Требуются изменения лишь в наборе параметров передаваемых функциям библиотеки стандартной периферии. Предполагается, что эту часть кода пользователь напишет самостоятельно в интегрированной среде разработки. (В текущей версии функционал кодогенерации работает в режиме демонстрации идеи алгоритмов экспорта данных).

Заметки автора

Актуальной является задача унификации имён выводов периферийных устройств микроконтроллеров. Её нужно решать в отношении микроконтроллера К1986ВК01GI, у которого таблица спецификации о функциональном назначении ножек содержит более 1500 позиций. У микроконтроллера К1921ВК01Т этот список в 3 раза короче, но его можно рекомендовать для проверки решения. Имена функций, оптимизированные для К1986ВК01GI должны подойти К1921ВК01Т.

В спецификации на инженерную версию микроконтроллера К1986ВК01GI [1] сотни функций были неудобны для восприятия и неоптимальны для автоматизированной обработки. Автор составил версии программы для десятка микроконтроллеров разных производителей, и в каждом случае обходил обозначенное затруднение простым решением. Выше было сказано – имена выводов периферийных устройств, перечислены в csv-файлах, которые внедрены в веб-страницы. После запуска скрипты меняют содержимое этих файлов с помощью регулярных выражений. Своим волевым решением автор изменил и унифицировал имена функций у всей совокупности микроконтроллеров, для которых написал программу. После обращения в службу поддержки, команда программистов АО «ПКК Миландр» рассмотрела предложения, и, с первой итерации, унифицировала 80% имён выводов периферийных устройств.

Приложение. Правила создания имён для обозначения функционального назначения ножки микроконтроллера

Сформулируем правила создания имён для обозначения функционального назначения ножки микроконтроллера. Имена функций должны включать корень (USART, CAN), позволяющий выбрать всю группу выводов того или иного периферийного устройства за одну проверку. Например, имена функций: SCL, SDA, TX_EN, nTZ1_EXT, – корня не имеют. Корни не должны содержать цифру. Неудачными именами являются I2Cx_SCL и I2Cx_SDA. Цифра в корне может мешать разделению группы по номеру в суффиксе. Лучше использовать альтернативный корень IIC вместо I2C. Для обращения к устройству ввода вывода на языке Си успешно используют его имя и порядковый номер (USART1, USART2). Поэтому корень имени функции может быть дополнен цифрой-суффиксом (TMR[1]_CH1P), но не цифрой-окончанием (TMR_CH1P_[1]). Нумерация устройств должна начинаться с единицы (человеку необходимо, чтобы последний порядковый номер совпадал с их количеством). В масштабе микроконтроллера идея использования приставок для обозначения входа устройства, [n]RST, является сомнительной. Имена функций не должны содержать приставок ([CH]11_ADC1, [TX]_CANFD, [n]UART) – выборка групп функций должна выполняться по совпадению с маской (с корнем) от начала строки. В противном случае алгоритм фильтрации усложняется для разделения групп CH11_ADC1 и TMR_CH1P_1. В языке Си имя константы не может включать арифметические знаки. Поэтому недопустимы имена CMP_IN+ и CMP_IN-. Следует использовать CMP1_InP и CMP1_InN (или CMP1_INP, CMP1_INM). В этом контексте рекомендуется пересмотреть имена TMR_CH1P_1 в пользу шаблона TMR1_CH1P. Из-за асимметрии по высоте в начертания букв "p" и "n" в нижних регистрах их лучше не использовать. Правила применения близких по смыслу окончаний требуют пересмотра: [P]-plus / [M]-minus, [P]-positive / [N]-negative, [A] / [B], [T]-top / [B]-bottom, – поскольку таймеры в современных микроконтроллерах дифференцировались по назначению (QEP, CAP, PWM). Предлагается проверить обоснованность существования двух версий суффикса TX и TXD. Можно подумать об альтернативном имени устройства CANFD. Поскольку корень CAN не позволяет разделить устройства CAN и CANFD. В последнюю очередь следует унифицировать имена: EXTRST, GND, DGND, AGND, TAGND, GNDBAT, GNDPOR, GNDUSB, VDD33, VDD12, – и другие их версии.
