# Исходники библиотек libc и libm
Точка входа после сброса находится в файле src/libc/crt0.S (reset_handler).  
Производиися настройка регистров SP и GP (указатель стека и "Global Pointer").  
Устанавливается точка входа для прерываний на 0x20000C0 (в ОЗУ, начало секции ".trap_handler_ram").  
Содержимое секций .data* копируется на своё место в ОЗУ (кроме случая сборки для запуска из ОЗУ).  
Заполняется нулями область переменных, которые при старте должны иметь значение ноль.  
Затем вызываются последовательно функции:  
1. *void __libc_init_mcu()* - настраивает источник системного тактирования на генератор с внешним кварцем, ожидается, что частота кварца 32 МГц. Настраивается системный таймер на счёт с частотой 32 кГц и срабатывание "будильника" каждую миллисекунду. И в завершение вызывается функция *void process_init_mcu(void)*, в которой по-умолчанию глобально включаются прерывания, и эту функцию можно переопределить в своём коде.  
2. *void __libc_init_array()* - если используется код на C++, здесь вызываются конструкторы глобальных объектов.  
3. *void main()* - собственно точка входа в основной код.  
Обработчик прерываний *void trap_handler(void)* (src/libc/runtime.c) всегда размещается в ОЗУ и выполняет следующие действия:  
1. Вызывает функцию *bool process_irqs(void)*, которая по-умолчанию только возвращает **false**.  Эту функцию можно переопределить в своём коде для обработки прерываний от оборудования микроконтроллера. Если вернуть **true**, то обработчик прерывания после этого сразу завершится.  
2. Если вызов *bool process_irqs(void)* вернул **false**, то производится проверка наличия ожидающего прерывания от системного таймера и его обработка при необходимости.  
Соответственно, функцию *bool process_irqs(void)* тоже лучше расположить в ОЗУ для более быстрой и предсказуемой обработки прерываний.
