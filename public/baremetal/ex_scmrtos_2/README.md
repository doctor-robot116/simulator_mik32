# Пример работы с scmRTOS
- В качестве системного использован собственно системный таймер, прерывания от которого идут с частотой 1000 Гц.
- Таймер TIMER32_0 настроен на частоту примерно 1000.1 Гц.
- В обработчике прерывания от таймера TIMER32_0 читается результат преобразования от АЦП и передаётся в событии msgAdcValue вместе с номером канала. На самом деле между обнаружением наличия события и собсвенно копированием значения события может вклиниться обработчик прерывания и подсунуть уже очередное сообщение в объект синхронизации.
- Процесс (PPrinter) ждёт сообщения (msgAdcValue), при получении сообщения в UART_0 выводится номер канала и значение замера из полученного сообщения.
- Процесс (PLedEventGen) "взводит" событие ledEvent и спит 499 тиков системного таймера.
- Процесс (PLedSwitcher) ожидает события ledEvent, как дождался - меняет состояние выхода PORT2.7 (на котором висит светодиод).
- Включено 4 канала АЦП - с номерами 0, 1, 3 и 4. Замеры проводятся циклически по этим каналам.

***

## Подключение:
- Канал 0 АЦП - PORT1.5
- Канал 1 АЦП - PORT1.7
- Канал 3 АЦП - PORT0.4
- Канал 4 АЦП - PORT0.7
